-----
用戶註冊(/register)只由有權限者協助添加
只需要寫username和role，預設密碼和username相同
使用者登入後如果email為null必須要先填，補完資料(/update)才可正常操作
重置密碼(/resetPassword)方式是寄信到mail，包含隨機字串密碼
重置密碼後登入必須更新密碼(/updatePassword)才能繼續動作
sidebar按順序
1.user用戶管理:
(/list)模糊查詢姓名/id，查分頁，排序
(/register)協助新增用戶，填username和role
(/update)更新用戶，主要用來更新role[非本人即通知用戶]
(/clientLock)操作用戶鎖定
(/clientStatus)操作用戶狀態
2.權限管理role(permission+components(Router))
role:
(/list)清單
(/update)更新角色名稱
(/add)新增角色，預設是沒permission
(/remove)移除角色
(/rolePermission)更新角色權限
(/roleRouter)更新角色可訪問路由
permission:
(/list)後端功能權限清單，簡單分類，不分層
(/role)針對role拿permissionIdList
(/ban)禁用
Router:
(/configList)前端路由配置列表，刪去分層
(/list)展示清單，有整理分層
(/role)針對role拿routerIdList
Department:
(/list)部門清單:
(/staff)部門人員
(/edit)新增/編輯部門
(/remove)移除部門，使用中不可移除
3.薪資管理salary
管理層:
(/roots)薪資基礎清單
(/rootBy)特定用戶薪資基礎表
(/editRoot)編輯或新增薪資基礎表[編輯即通知用戶]
展示:
(/get)個人薪資清單
(/info)特定薪資詳細	
[每月結算通知用戶]
4.績效管理performance
(/allList)所有績效紀錄
(/list)個人績效紀錄
(/add)新增績效[通知主管級用戶過審]
(/update)更新績效紀錄
(/remove)移除績效-非未審不可刪
(/accept)過審[通知用戶]
5.打卡簽到attend
(/signIn)上班
(/signOut)下班
6.休假管理leave
(/pendingList)待審核假單列表
(/list)個人假單列表
(/add)新增假單[通知主管級用戶過審]
(/update)更新假單
(/delete)刪除假單-非未審不可刪
(/accept)過審[通知用戶]
7.專案管理project
(/list)專案清單
(/add)新增專案
(/update)更新專案
{/start)開案
{/done)結案
8.採購管理procurement
(/list)清單
(/add)新增一筆採購
(/update)編輯採購內容
9.任務job
(/list)所有任務，依狀態分類
(/add)新增任務，可搜尋(/client/nameList)指派人員
(/update)編輯任務，可搜尋(/client/nameList)指派人員
(/delete)刪除任務
10.排程quartz_job(和上面任務無關，是quartz的排程，做一對一，只用來做簡單控制)
(/list)現有排程清單、狀態
(/add)新增排程，程式碼要先加上job實現，新增時要帶上jobClass路徑
(/update)更動排程時間cron、狀態
(/delete)刪除排程(job+trigger)
(/toggle)切換排程狀態(暫停/開始)
(/exec)單次執行排程任務
